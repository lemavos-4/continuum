# Continuum ‚Äî Offline-First

The frontend now ships an offline-first layer so the app keeps working without internet and syncs back automatically when connection returns.

## What works offline

- **App shell**: the SPA loads with no network (Service Worker + cached HTML/CSS/JS/fonts).
- **Browsing previously-loaded data**: every successful `GET /api/...` response is mirrored into IndexedDB. When offline, the same request returns the cached payload transparently.
- **Mutations** (create / update / delete) for notes, entities, folders, graph edges, activities, time-tracking sessions, account preferences and wallpaper metadata: the request is queued in IndexedDB (`sync_queue`) and the UI gets an optimistic `202` response so it never blocks.
- **Wallpaper image**: the binary blob is persisted in IndexedDB the first time it's fetched, so it still displays offline.

## What still requires network

- Login / register / OAuth / token refresh.
- File uploads (multipart) ‚Äî they're not safely replayable as JSON.
- Subscription/checkout and bulk imports.

## Conflict resolution

Pure Last-Write-Wins by `updatedAt`. Every queued mutation carries the `X-Client-Updated-At` header (epoch ms) so the backend can compare with the persisted record. If timestamps tie the server wins. Deletes participate in LWW like any other update via soft-delete tombstones ‚Äî extend each entity's `DELETE` handler to set `deleted = true` and bump `updatedAt` rather than removing rows, and include tombstones in list responses so other devices see the deletion.

## Sync UX

- Status pill in the top-right (desktop) and mobile top bar:
  - `Online` (hidden when nothing is pending)
  - `Offline ¬∑ N pending`
  - `Syncing‚Ä¶`
  - `Sync issue ¬∑ N pending`
- Profile ‚Üí Preferences ‚Üí **Sync now** button forces a queue flush.
- Automatic flush triggers: app boot (if online), `online` event, every 60s while online.

## Service Worker safety

- Generated by `vite-plugin-pwa` (`generateSW`, `registerType: "autoUpdate"`).
- Registered only from `src/lib/pwa-register.ts`, which refuses dev, iframes, `?sw=off`, and any `*.lovableproject.com` / `*.lovable.app` / `*.beta.lovable.dev` preview host.
- Navigation strategy: `NetworkFirst` with `/index.html` fallback. `/api/**` and `/~oauth*` are excluded from navigation fallback.
- Auth headers are never touched by the SW ‚Äî axios attaches them in JS.

## Backend follow-ups (recommended)

The frontend is fully ready. The backend should adopt these conventions to make LWW deterministic across devices:

1. Add `deleted: boolean` (default false) and bump `updatedAt` on every write to: `Note`, `Entity`, `EntityLink`, `Activity`, `TimeSession`.
2. Treat `DELETE /api/<entity>/{id}` as soft delete (`deleted=true`, bump `updatedAt`).
3. On `PUT/PATCH`, read the `X-Client-Updated-At` header; if older than the server's `updatedAt`, return the current server record unchanged (server wins).
4. Accept `?since=<epochMs>` on list endpoints to return only deltas (including tombstones).

Until those land, the existing endpoints still work ‚Äî the queue just replays operations FIFO and the server's natural ordering acts as the tiebreaker.
